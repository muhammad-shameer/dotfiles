## Overview

These dotfiles configure a fully themed Wayland desktop built around:

| Component | Role |
|-----------|------|
| **Hyprland** | Tiling Wayland compositor |
| **Kitty** | GPU-accelerated terminal emulator |
| **Neovim** | Text editor with LSP, AI completion, and fuzzy finding |
| **Waybar** | Status bar |
| **Wofi** | Application launcher |
| **SwayNC** | Notification center |
| **Pywal** | Dynamic color theming from wallpaper |
| **Hyprpaper** | Wallpaper daemon |
| **Hypridle** | Idle/lock daemon |
| **Hyprlock** | Lock screen |

Colors are sourced from `~/.cache/wal/colors-hyprland.conf` ‚Äî generated by pywal from the current wallpaper ‚Äî and applied globally across Hyprland, Kitty, and Hyprlock.

## Prerequisites

Install the following packages before deploying these dotfiles. On Arch Linux:

```bash
# Core compositor
sudo pacman -S hyprland hyprpaper hypridle hyprlock

# Terminal & launcher
sudo pacman -S kitty wofi

# Status bar & notifications
sudo pacman -S waybar swaync

# Theming
sudo pacman -S python-pywal

# Neovim
sudo pacman -S neovim git

# Font (required ‚Äî used everywhere)
sudo pacman -S ttf-firacode-nerd

# Optional but used in configs
sudo pacman -S dolphin playerctl brightnessctl wpctl
```

> **Note:** If you use an AUR helper like `yay` or `paru`, replace `sudo pacman -S` with `yay -S` for any packages not in the official repos.

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/muhammad-shameer/dotfiles.git ~/dotfiles
cd ~/dotfiles
```

### 2. Back up existing configs

```bash
cp -r ~/.config/hypr ~/.config/hypr.bak 2>/dev/null
cp -r ~/.config/kitty ~/.config/kitty.bak 2>/dev/null
cp -r ~/.config/nvim ~/.config/nvim.bak 2>/dev/null
cp -r ~/.config/waybar ~/.config/waybar.bak 2>/dev/null
cp -r ~/.config/wofi ~/.config/wofi.bak 2>/dev/null
cp -r ~/.config/swaync ~/.config/swaync.bak 2>/dev/null
```

### 3. Copy configs

```bash
cp -r .config/hypr     ~/.config/hypr
cp -r .config/kitty    ~/.config/kitty
cp -r .config/nvim     ~/.config/nvim
cp -r .config/waybar   ~/.config/waybar
cp -r .config/wofi     ~/.config/wofi
cp -r .config/swaync   ~/.config/swaync
cp -r .config/wal      ~/.config/wal
cp -r .config/wallpapers ~/.config/wallpapers
```

### 4. Generate initial pywal colors

Pywal must be run once to create `~/.cache/wal/` ‚Äî required by Hyprland, Kitty, and Hyprlock on first launch:

```bash
wal -i ~/.config/wallpapers/vibrant-train.jpg
```

To use your own wallpaper:

```bash
wal -i /path/to/your/wallpaper.jpg
```

### 5. Make waybar scripts executable

```bash
chmod +x ~/.config/waybar/scripts/*.sh
chmod +x ~/.config/swaync/refresh.sh
```

### 6. Launch Hyprland

Start a Hyprland session from your display manager or TTY:

```bash
Hyprland
```

## Components

### Hyprland

**Config:** `~/.config/hypr/hyprland.conf`

The window manager uses the **dwindle** layout with:

- **Monitor:** 1920√ó1080 @ 144Hz (auto-detected)
- **Gaps:** 5px inner, 20px outer
- **Border size:** 1px, colored from pywal (`$color9` active, `$color5` inactive)
- **Rounding:** 10px corners
- **Blur:** enabled (size 6, 2 passes) on windows, waybar, wofi, and swaync
- **Animations:** Smooth bezier curves on windows, workspaces, borders, and layers

**Default applications:**

| Variable | App |
|----------|-----|
| `$terminal` | `kitty` |
| `$fileManager` | `dolphin` |
| `$menu` | `wofi --show drun -n` |

**Autostart:** `hyprpaper` and `waybar` launch automatically on login.

---

### Kitty

**Config:** `~/.config/kitty/kitty.conf`

- **Font:** FiraCode Nerd Font, size 16pt
- **Background opacity:** 0.9 (slight transparency)
- **Color scheme:** dynamically loaded from `~/.cache/wal/colors-kitty.conf` (pywal)

No custom keybindings are set ‚Äî all kitty defaults apply.

---

### Waybar

**Config:** `~/.config/waybar/config` and `~/.config/waybar/style.css`

The bar sits at the **top** of the screen with three zones:

| Zone | Modules |
|------|---------|
| Left | Clock (HH:MM:SS AM/PM), System tray |
| Center | Hyprland workspaces (icons, 5 persistent) |
| Right | Notification bell (SwayNC toggle) |

Additional modules are defined but not active in the default layout: network, bluetooth, battery, CPU, memory, temperature, color picker, and a pacman update counter.

**Themes:** Four Waybar visual themes are available in `~/.config/waybar/themes/`:
- `default`
- `experimental`
- `line`
- `zen`

Switch themes via: `Super + R` (runs `~/.config/waybar/scripts/launch.sh`).

---

### Wofi

**Config:** `~/.config/wofi/config`

Application launcher with:

- **Mode:** `drun` (desktop app search)
- **Size:** 500√ó600px
- **Images:** enabled (32px icons)
- **Case-insensitive** search
- **Terminal:** `kitty`

Launch with: `Super + Space`

---

### SwayNC

**Config:** `~/.config/swaync/config.json`

Notification center positioned at the **bottom-right**. The control panel contains:

- MPRIS media player widget
- Notification list
- Volume slider (wpctl)
- Brightness slider
- Quick-action buttons: mute toggle, network (nmtui), DND toggle, lock screen, reboot, shutdown

Toggle the panel: click the notification bell in Waybar, or use `Super + R` Waybar shortcut.

---

### Pywal

**Config:** `~/.config/wal/templates/colors-hyprland.conf`

Pywal generates a color palette from your wallpaper and writes it to `~/.cache/wal/`. All components source colors from there automatically.

**To change the wallpaper and regenerate colors:**

```bash
wal -i /path/to/new-wallpaper.jpg
# Then reload Hyprland config:
hyprctl reload
# And restart waybar:
~/.config/waybar/scripts/launch.sh
```

## Customization

### Change the wallpaper

```bash
wal -i /path/to/image.jpg
hyprctl reload
~/.config/waybar/scripts/launch.sh
```

### Change monitor resolution or refresh rate

Edit `~/.config/hypr/hyprland.conf`:

```ini
monitor=,1920x1080@144,auto,auto
# Format: monitor=NAME,WIDTHxHEIGHT@HZ,POSITION,SCALE
```

### Change the default terminal, file manager, or menu

Edit `~/.config/hypr/hyprland.conf`:

```ini
$terminal = kitty        # change to alacritty, foot, etc.
$fileManager = dolphin   # change to thunar, nautilus, etc.
$menu = wofi --show drun -n
```

### Change Waybar theme

Run the select script or press `Super + R` to cycle themes. To set one permanently, edit `~/.config/waybar/scripts/launch.sh`.

### Adjust idle/lock timers

Edit `~/.config/hypr/hypridle.conf`. Timeouts are in seconds:

```ini
listener {
    timeout = 100    # screensaver
    ...
}
listener {
    timeout = 1000   # lock screen
    ...
}
listener {
    timeout = 2400   # display off
    ...
}
```

### Neovim: add a new LSP server

Open Neovim and run `:Mason` to browse and install servers. Then add the server name to `~/.config/nvim/lua/plugins/lsp.lua` under `ensure_installed` and the `servers` table.

## Troubleshooting

**Colors are missing / Hyprland shows warnings about pywal**

Run `wal -i ~/.config/wallpapers/vibrant-train.jpg` to generate the cache files, then restart Hyprland.

**Neovim plugins not loading**

On first launch, lazy.nvim will automatically clone all plugins. If that fails, run `:Lazy sync` inside Neovim.

**Waybar not starting**

Check that the scripts are executable: `chmod +x ~/.config/waybar/scripts/*.sh`. Then restart with `~/.config/waybar/scripts/launch.sh`.

**Hyprlock shows wrong monitor name**

Edit `~/.config/hypr/hyprlock.conf` and replace `HDMI-A-1` with your actual monitor name. Find it with: `hyprctl monitors`.

**Wofi not launching**

Ensure wofi is installed (`which wofi`) and the keybind in `hyprland.conf` matches your `$menu` variable.

**Font icons not rendering**

Install FiraCode Nerd Font: `sudo pacman -S ttf-firacode-nerd` (Arch) or download from [nerdfonts.com](https://www.nerdfonts.com/).
